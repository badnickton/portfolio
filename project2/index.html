Airbnb Database Project

Database Creation and Management

About This Project

This project focuses on creating a scalable and robust database for Airbnb, a global marketplace for lodging and travel experiences. Our system supports essential functionalities, such as managing properties, renters, bookings, payments, and reviews while ensuring privacy and reliability.

Project Steps

Step 1: Initial Planning

The first step was to identify the key entities and relationships needed to model the database. The main entities included:

  • Renter/Guest: Individual who books properties through Airbnb for short-term stays.
  • Property: Details about rental properties available for booking.
  • Booking: Records details about reservations, including dates, guests, and payment status.
  • Review: Feedback from hosts and guests regarding stays and interactions.
  • Payment: Handles payment transactions, including booking charges and refunds.
  • Message: Manages communication between hosts and guests within the platform.
  • Landlord: Owns/manages the property being rented out.

We mapped out the necessary attributes for each entity and defined how they would relate to each other.

Step 2: Database Design

In the second step, we created an Entity-Relationship (ER) diagram, which visually depicted the relationships between all the identified entities. This diagram helped us understand the structure of the database, ensuring that relationships like "a property belongs to a landlord" and "a booking is linked to a property" were represented correctly. This design also facilitated the avoidance of data redundancy.

Step 3: Data Type Selection

After designing the database structure, we carefully selected the appropriate data types for each attribute. For example, we used DECIMAL for amounts (like payments), TEXT for reviews, and DATE for dates related to bookings and reviews. This ensured data was stored efficiently and accurately.

Step 4: Creating the Database

With the design and data types in place, we proceeded to create the tables in SQL. This step involved writing SQL statements to define each table, its columns, and the relationships between them. Foreign keys were implemented to link entities, ensuring that related data was consistent. After creating the tables, we populated the tables by generating mock data from Mockaroo data generator.

Step 5: Query Development

Once the tables were created, we wrote SQL queries to interact with the database. These queries were designed to retrieve relevant information, such as booking details, landlord information, or reviews. The queries were also used to insert, update, and delete data as necessary.

Example Query

CREATE OR ALTER PROCEDURE GetTopLandlordsByRent AS BEGIN SET NOCOUNT ON; SELECT TOP 5 L.LandlordID, L.Name AS LandlordName, SUM(B.AmountPaid) AS TotalRentCollected FROM Landlord L JOIN Property P ON L.LandlordID = P.LandlordID JOIN Booking B ON P.PropertyID = B.PropertyID GROUP BY L.LandlordID, L.Name ORDER BY TotalRentCollected DESC; END;

This query identifies the top 5 landlords based on total rent collected, helping property managers recognize significant contributors and forecast revenue.

Step 6: Finalizing the Application

We finalized everything by ensuring that no data was missing, all the queries made sense, and that everything was realistic for a presentation as a data scientist at Airbnb.
